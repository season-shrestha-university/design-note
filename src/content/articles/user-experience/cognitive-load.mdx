Every time a user interacts with an application, they expend mental energy. They have to decode the navigation, remember what icon means what, and process the actual information they came for. The sum of this mental effort is known as cognitive load. When cognitive load exceeds a user's working memory capacity, they become frustrated, make errors, and often abandon the task entirely.

UX designers must aggressively manage cognitive load. Intrinsic cognitive load is the inherent difficulty of the task itself (e.g., filing taxes is inherently complex). However, extraneous cognitive load is the effort required to deal with a confusing interface. 

We can reduce extraneous load by relying on established design patterns so users don't have to learn new systems. Using progressive disclosure—revealing advanced options only when requested—keeps the initial interface clean. Breaking long forms into multiple, bite-sized steps transforms an intimidating wall of inputs into an effortless flow. A well-designed app should feel like it does the thinking for you.
